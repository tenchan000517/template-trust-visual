# QAチェックリスト

**このドキュメントは、品質チェック工程を担当するエージェントが読むもの。**
**目的: 完成したページを複数の観点から検証し、修正すべき箇所を報告する。**

**設定は `docs/content-config.md` を参照すること。**

---

## このガイドの使い方

1. 実装済みのコンテンツ（Markdownファイル + データ登録）を受け取る
2. **content-config.md を読み、品質基準・ブランドボイス・実装チェック項目を確認する**
3. 以下のチェックリストを全て実行する
4. 不合格の項目は「不合格項目一覧」にまとめて報告する
5. **1つでも不合格なら修正を要求する。「まあいいか」は許されない**

---

## チェック1: 品質基準（content-config.md 参照）

content-config.md の品質基準に**全てYes**と言えなければ不合格。

1. ターゲットが読んだ後に「次に何をすべきか」わかるか？
2. 自社の強み・哲学が自然に体現されているか？
3. 「どこにでもありそうな一般論」になっていないか？
4. 読者がCTAに進みたくなる感情の流れがあるか？
5. 全ての数値に出典があるか？

---

## チェック2: コンテンツ品質

### 行動可能性
- [ ] 読んだターゲットが「次に何をすればいいか」わかるか
- [ ] 手順が具体的か（抽象的な「考え方」で終わっていないか）

### 正確性
- [ ] 全ての数値に出典（調査名・発表元・年度）があるか
- [ ] 推測表現（「一般的に」「よく言われる」「〜でしょう」）が含まれていないか
- [ ] 過去の日付を未来形で書いていないか
- [ ] 数字が文脈に埋め込まれているか（「○○%」で終わっていないか）

### ブランドボイス
- [ ] content-config.md の口調ルールに沿っているか
- [ ] content-config.md のNGルールに抵触していないか
- [ ] テンプレ文言（「お客様に寄り添い」「成長できる環境」等）が使われていないか

---

## チェック3: Markdown品質

- [ ] frontmatterが完全か（title, date, category, excerpt, tags, readingTime, image）
- [ ] excerptが120文字以内か（meta description用）
- [ ] 見出し階層が適切か（h2 → h3の順。h1は使わない）
- [ ] リスト・引用・太字が正しく使われているか

---

## チェック4: データ登録

content-config.md の「実装時に更新するファイル一覧」を全てチェックする。

- [ ] Markdownファイルが指定パスに配置されているか
- [ ] データ定義ファイルにカード情報が追加されているか
- [ ] データ定義ファイルで `available: true` になっているか
- [ ] サムネイル画像が指定パスに存在するか
- [ ] sitemap に反映されているか（動的 or 手動追加）
- [ ] `ai.txt` にコンテンツ情報が追記されているか
- [ ] `llms.txt` にコンテンツ情報が追記されているか
- [ ] `docs/インデックス登録リクエスト手順書.md` にURLが追記されているか（ファイルがある場合）

---

## チェック5: SEO

- [ ] excerptがmeta descriptionとして適切か（120文字以内、キーワード含む）
- [ ] tagsがSEOキーワードと対応しているか

---

## チェック6: 禁止事項

以下が1つでもあれば不合格。

- [ ] bounceアニメーションが使われていないか
- [ ] 同色の薄いテキスト（`text-white/70`等）がないか
- [ ] 絵文字アイコンがないか
- [ ] テンプレ的なCTA文言（「あなたの〇〇を応援します」等）がないか

---

## 報告フォーマット

```markdown
# QAチェック結果: [ページタイトル]

## ステータス: 合格 / 不合格

## 不合格項目一覧
（番号付きリスト。各項目に「チェック番号」「具体的な問題」「修正案」を記載）

1. **[チェック2: 正確性]** セクション3の統計に出典がない
   → リサーチ結果から出典を追記

## 合格項目のハイライト
（特に品質が高かった点を1-2個挙げる）
```

---

## QA完了後

全チェック合格したら、**content-config.md の「コンテンツ生成履歴」テーブルに追記する。**

```
| [slug] | [タイトル] | [カテゴリ] | [使用アンカー] | YYYY-MM-DD |
```
