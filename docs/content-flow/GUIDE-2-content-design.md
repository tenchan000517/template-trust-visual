# コンテンツ設計ガイド

**このドキュメントは、構成設計工程を担当するエージェントが読むもの。**
**目的: リサーチ結果を受け取り、誰に・何を・どの順で伝えるかを設計する。**

**設定は `docs/content-config.md` を参照すること。**

---

## このガイドの使い方

1. リサーチ済みデータを受け取ったら、**まず `docs/content-config.md` を読む**
2. ターゲット定義・品質基準・構成パターンを確認する
3. このガイドのルールに従って構成案を設計する
4. 構成案をユーザーに提示し、承認を得てから次工程へ渡す

**判断に迷ったらユーザーに聞く。「おそらくこうだろう」で進めない。**

---

## 設計原則

- **読んだターゲットが「次に何をすればいいか」わかること。** これが唯一の品質基準
- **「どこにでもある一般論」は出す価値がない**
- **出典のない情報は存在しないものと同じ**
- **感情の流れを設計する。** 情報を並べただけでは人は動かない

---

## 構成設計の手順

### Step 1: ターゲットを確認する

content-config.md の「ターゲット定義」を読み、**このページは誰の、どんな悩みに応えるか** を明確にする。

### Step 2: 「読んだ後に『できる』こと」を1文で書く

これが構成案の最重要項目。曖昧なら構成全体がぶれる。

例:
- OK: 「リフォーム業者の選び方を3つの基準で判断でき、見積もり依頼ができる」
- NG: 「リフォームの大切さがわかる」（← 抽象的すぎる。何が「できる」のか不明）

### Step 3: 構成パターンを選択する

content-config.md の「構成パターン」から最適なものを選ぶ。
パターンに当てはまらない場合はユーザーに相談する。

### Step 4: セクション構成を設計する

各セクションに「目的」と「内容概要」を明記する。**目的のないセクションは削除する。**

### Step 5: 感情の流れを確認する

4段階の感情設計:
1. **共感** — 「わかる、自分もそう思ってた」
2. **気づき** — 「そういう見方があったのか」
3. **希望** — 「自分にもできるかもしれない」
4. **行動** — 「やってみよう / もっと知りたい」

### Step 6: 品質基準の問いに全てYesと言えるか確認する

## 構成案の出力フォーマット

```
## 構成案: [ページタイトル]

**カテゴリ**: [カテゴリ]
**URL**: [hub_url]/[slug]
**使用パターン**: マインドセット型 / ハウツー型 / 事例型

### ターゲット
[誰の、どんな悩みに応えるか]

### 読んだ後に「できる」こと
[1文で]

### SEOキーワード
[メイン1 + サブ2-3]

### セクション構成

| # | セクション名 | 目的 | 内容概要 | 感情段階 |
|---|------------|------|---------|---------|
| 1 | ヒーロー | 共感 | タイトル案 + サブテキスト案 | 共感 |
| 2 | ... | ... | ... | 気づき |

### 出典の使い方
[リサーチ結果のどのデータをどのセクションで使うか]

### CTA設計
[どの感情ポイントでどのCTAを配置するか]

### 特記事項
[このページ固有の注意点]
```
