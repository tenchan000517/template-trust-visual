# コンテンツ生成フロー

**「コンテンツを作成してください」で動く5段階パイプライン。**

---

## フロー概要

```
テーマ決定
    ↓
[GUIDE-1] リサーチ     ← WebSearch/WebFetchで裏どり
    ↓
[GUIDE-2] 構成設計     ← ユーザー承認
    ↓
[GUIDE-3] 執筆         ← Markdown出力
    ↓
[GUIDE-4] 実装         ← ファイル配置 + 画像生成 + データ登録
    ↓
[GUIDE-5] QA           ← 全チェック通過で公開
```

---

## 前提: 設定ファイル

**全工程が `docs/content-config.md` を参照する。** このファイルがなければ `/create-content` コマンドでヒアリングを実行して生成する。

---

## ドキュメント一覧

| ファイル | 役割 |
|---------|------|
| `docs/content-config.md` | **全工程が参照する設定ファイル。** ターゲット・ブランドボイス・品質基準・カテゴリ・画像生成ルール |
| `GUIDE-1-research.md` | リサーチ工程。テーマについて事実を集め、裏どりし、出典を記録 |
| `GUIDE-2-content-design.md` | 構成設計工程。セクション構成・感情設計・CTA配置を設計 |
| `GUIDE-3-writing.md` | 執筆工程。Markdown形式でコンテンツを書く |
| `GUIDE-4-implementation.md` | 実装工程。ファイル配置・画像生成・データ登録 |
| `GUIDE-5-qa.md` | QA工程。品質チェック・修正要求 |

---

## content-config.md のフォーマット

ヒアリング完了後に生成される設定ファイルの構造:

```markdown
---
project_name: "[プロジェクト名]"
site_url: "[URL]"
content_name: "[コンテンツ名]"
hub_url: "[ハブページURL]"
created: "YYYY-MM-DD"
---

## 品質基準

コンテンツが完成したら、以下の問いに全てYesと言えること:

1. ターゲットが読んだ後に「次に何をすべきか」わかるか？
2. 自社の強み・哲学が自然に体現されているか？
3. 「どこにでもありそうな一般論」になっていないか？
4. 読者がCTAに進みたくなる感情の流れがあるか？
5. 全ての数値に出典があるか？

## 構成パターン

### パターン1: マインドセット型
1. フック（読者の悩み・問いかけ）
2. 視点の転換（新しい見方を提示）
3. 自社の事例 or 具体的な行動指針
4. まとめ（次の一歩を明確に）
5. CTA

### パターン2: ハウツー型
1. 課題の提示
2. 解決のフレームワーク（3-5ステップ）
3. 各ステップの詳細（データ or 事例付き）
4. よくある失敗パターン
5. まとめ + CTA

### パターン3: 事例型
1. 課題・状況の提示
2. 自社の具体的な事例・エピソード
3. そこから導かれる原則
4. 読者の状況への応用
5. CTA

## CTA・導線

### メインCTA（各コンテンツ末尾）
- **[CTA文言]** → [URL]

### サブ導線
- 関連コンテンツへのリンク
- [その他の導線]

### 導線の入れ方
- 感情が高まったポイントでCTAを置く。機械的に末尾に置くだけでは効果薄い
- CTA文言はページの文脈に合わせて微調整する（テンプレ文言の使い回し禁止）

## URL・配置

```
[hub_url]/              ← ハブページ（カード一覧）
[hub_url]/[slug]        ← 個別コンテンツページ
```

### ファイル構成
- **Markdown記事**: src/content/[content_name]/[slug].md
- **カード定義**: src/app/[hub_url]/[content_name]-data.ts
- **ハブページ**: src/app/[hub_url]/page.tsx
- **個別ページ**: src/app/[hub_url]/[slug]/page.tsx (SSG)

## 実装時に更新するファイル一覧

| ファイル | 更新内容 |
|---------|---------|
| src/content/[name]/[slug].md | 記事本体 |
| [データ定義ファイル] | カード情報追加、available: true |
| public/img/[name]/[slug].png | サムネイル画像 |
| src/app/sitemap.ts | URLエントリー追加（動的なら不要） |
| public/ai.txt | コンテンツ情報追記 |
| public/llms.txt | コンテンツ情報追記 |
| docs/インデックス登録リクエスト手順書.md | URL追記（ファイルがある場合） |
