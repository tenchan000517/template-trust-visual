# コンテンツ生成ワークフロー

「コンテンツを作成して」「記事を追加して」等の指示で起動する。

## 起動時の処理

### 1. `docs/content-config.md` を探す

- **存在する** → 読み込み、`docs/content-flow/README.md` のパイプラインに従って実行
- **存在しない** → ヒアリングシーケンスを開始

### 2. ヒアリングシーケンス（初回のみ）

`docs/content-config.md` がない場合、以下の質問を **1問ずつ順番に** ユーザーに聞く。
`docs/seo-config.md` が存在する場合、ターゲット定義はそこから引き継ぎ、該当の質問はスキップする。

```
Q1: プロジェクト名とサイトURLを教えてください。
    （seo-config.md があればスキップ）

Q2: ターゲットを教えてください。
    - 年齢層、属性、地域
    - 心理状態・悩み（例: 「リフォームしたいけど業者選びがわからない」）
    （seo-config.md があればスキップ）

Q3: コンテンツの名前を教えてください。
    （例: ブログ / コラム / ガイド / お役立ち情報 / スタッフブログ）

Q4: ハブページ（記事一覧ページ）のURLはどこにしますか？
    （例: /blog, /column, /guide, /useful-info）

Q5: カテゴリ体系を教えてください。
    決まっていなければ「未定」でOK — 業種とターゲットから提案します。
    （例: 「施工事例」「お役立ち情報」「業界ニュース」）

Q6: ブランドボイス（文章のトーン）はどうしますか？
    a) フォーマル（です/ます体・専門的）
    b) カジュアル（親しみやすい・語りかけ調）
    c) 専門的だけど読みやすい（データ重視・平易な表現）
    d) その他（具体的に）

    文章で絶対にやってはいけないこと（NGルール）はありますか？
    （例: 「！の多用禁止」「業界用語をそのまま使わない」）

Q7: 記事の末尾にどんなCTAを置きますか？
    （例: お問い合わせ / 資料請求 / エントリー / 無料相談）
    リンク先のURLも教えてください。

Q8: アンカー画像（画像スタイルの基準になる画像）はありますか？
    あればパスを教えてください。なければAI生成時にスタイルを一から作ります。
    ※ 画像は基本的にAI生成（Nanobanana使用）
```

### 3. 設定ファイル生成

全回答を得たら `docs/content-config.md` を生成する。
フォーマットは `docs/content-flow/README.md` に記載。

### 4. パイプライン実行

設定ファイル生成後（または既存の場合）、`docs/content-flow/README.md` を読み、
5段階パイプラインに従ってコンテンツを生成する。

```
docs/content-flow/
├── README.md                    ← 全体フロー概要（最初にここを読む）
├── GUIDE-1-research.md          ← リサーチ（裏どり）
├── GUIDE-2-content-design.md    ← 構成設計（ユーザー承認）
├── GUIDE-3-writing.md           ← 執筆（Markdown）
├── GUIDE-4-implementation.md    ← 実装（ファイル配置・画像生成・データ登録）
└── GUIDE-5-qa.md                ← 品質チェック
```
