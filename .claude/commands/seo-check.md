# SEO/LLMO コンテンツ戦略監査（汎用版）

プロジェクトのWebサイトに対して、包括的なSEO/LLMO監査を実行する。
技術SEOに加え、**ターゲットの検索行動・AI質問パターンに基づくコンテンツ戦略提案**を行う。

---

## Phase 0: 設定ファイルの確認

### 最初に `docs/seo-config.md` を探す

- **存在する** → 読み込んで Phase 2 へ進む
- **存在しない** → Phase 1（ヒアリング）を実行する

---

## Phase 1: 初回ヒアリングシーケンス

設定ファイルがない場合、以下の質問を**1問ずつ順番に**ユーザーに聞く。
全回答を得たら `docs/seo-config.md` を生成して保存する。

### 質問リスト

```
Q1: プロジェクト名とサイトURLを教えてください。
    （例: 株式会社〇〇 / https://example.com）

Q2: 業種・業態を教えてください。
    （例: 建設業、IT、士業、製造業、コンサルティング、飲食、美容 等）

Q3: このサイトの一番の目的は何ですか？
    （例: 問い合わせ獲得 / 採用強化 / ブランディング / EC販売 等）

Q4: ターゲットを教えてください。
    - 年齢層（例: 20代前半、30-40代、50代以上）
    - 属性（例: 経営者、求職者、主婦、エンジニア）
    - 地域（例: 全国、愛知県、名古屋市）
    - 心理状態・悩み（例: 「転職したいけど不安」「リフォームしたいけど業者選びがわからない」）

Q5: 競合サイトはありますか？（最大3つ、URLがあれば）

Q6: 現在のコンテンツ戦略を教えてください。
    - ブログ/マガジン/コラムの有無
    - コンテンツ管理方式（Markdown / CMS / なし）
    - 更新頻度（週1 / 月1 / 不定期 / なし）

Q7: SEO/LLMOで特に重視したいことはありますか？
    （例: Google検索上位 / AIに回答として拾われたい / 両方 / わからない）

Q8: 狙いたいキーワードがあれば教えてください。（なければ「なし」でOK）
    （例: 「名古屋 リフォーム」「20代 キャリア」「弁護士 相続 名古屋」）
```

### 設定ファイル生成

全回答を得たら、以下のフォーマットで `docs/seo-config.md` を作成する:

```markdown
---
project_name: "[プロジェクト名]"
site_url: "[URL]"
industry: "[業種]"
primary_goal: "[サイトの目的]"
created: "YYYY-MM-DD"
last_audit: ""
---

## 競合サイト

2. [URL or なし]
3. [URL or なし]

## コンテンツ戦略

- **ブログ/マガジンの有無**: [回答]
- **管理方式**: [回答]
- **更新頻度**: [回答]

- **重視する方向**: [回答]

## ターゲットキーワード

- [キーワード1]
- [キーワード2]
- [キーワード3]
（なしの場合は「未設定 — 監査時に調査」と記載）

## ターゲットのAI質問パターン（推定）

|------|--------|-----------|
```

設定ファイルを保存したら、「設定を保存しました。監査を開始します。」と伝えて Phase 2 へ進む。

## Phase 2: 並列監査（5エージェント同時起動）

`docs/seo-config.md` を読み込み、そこに書かれたターゲット・キーワード・方針に基づいて監査を実行する。

### Agent 1: 技術SEO監査
```
subagent_type: Explore

チェック項目:
- 全 page.tsx / layout.tsx のメタデータ（title, description, openGraph, twitter）
- canonical URL 設定
- 構造化データ（JSON-LD / Schema.org）の実装状況
  - Organization, WebSite, BreadcrumbList, Article, FAQPage, LocalBusiness, HowTo 等
- sitemap.ts/xml の網羅性（全公開ページが含まれているか）
- robots.txt の設定
- next.config 最適化（画像、圧縮、バンドル）
- Image コンポーネント最適化（sizes, priority, lazy）
- フォント最適化（display: swap, preload）
- セキュリティヘッダー（X-Frame-Options, HSTS, CSP 等）
- カスタム404ページの有無

出力: 技術的な問題リスト（Critical / Warning / Good）
```

### Agent 2: LLMO対策監査
```
subagent_type: Explore

チェック項目:
- robots.txt のAIクローラー許可設定
  （GPTBot, ChatGPT-User, Claude-Web, anthropic-ai, Applebot-Extended, PerplexityBot, Google-Extended, Bytespider）
- ai.txt の存在と内容品質
  - 全ページが網羅されているか
  - 説明が十分か（AIが文脈を理解できるか）
- llms.txt の存在と内容品質
  - LLMが参照した際に正確な情報を返せるか
- セマンティックHTML（article, section, nav, header, footer, time, figure）
- 見出し構造（h1→h2→h3 の階層が正しいか）
- パンくずリスト（BreadcrumbList 構造化データ）

出力: LLMO対策の充実度、AI回答に拾われるための改善案
```

### Agent 3: コンテンツ棚卸し
```
subagent_type: Explore

チェック項目:
- 全公開ページの一覧と各ページのコンテンツ量
- ブログ/マガジン記事の一覧と状態
- カテゴリ別の記事数バランス
- 各ページのSEOキーワード（meta description, tags）の網羅性
- CTA導線の設計（各ページからCVポイントへの接続）
- 内部リンク状況（孤立ページがないか）
- 画像のalt属性の設定状況

出力: コンテンツ棚卸し表（ページ/記事、状態、キーワード、内部リンク数、CTA有無）
```

### Agent 4: キーワード戦略調査
```
subagent_type: general-purpose（WebSearch使用）
※ フォアグラウンドで実行すること（バックグラウンド禁止）

docs/seo-config.md のターゲット定義・キーワードを参照して調査:

1. メインキーワードの検索トレンド・競合度調査
   - 設定済みキーワード + 業種×地域の組み合わせ
   - Google サジェスト・関連キーワードを収集

2. ロングテールキーワード発掘
   - ターゲットの悩み × 業種の掛け合わせ
   - 「〇〇 とは」「〇〇 方法」「〇〇 比較」系

3. 競合コンテンツ調査
   - 競合サイト（設定済みの場合）のコンテンツ傾向
   - 上位表示されているコンテンツの特徴
   - 競合が扱っていて自サイトが扱っていないテーマ

出力: キーワードリスト（メイン/ロングテール）、競合ギャップ分析
```

### Agent 5: AI質問パターン調査
```
subagent_type: general-purpose（WebSearch使用）
※ フォアグラウンドで実行すること（バックグラウンド禁止）

docs/seo-config.md のターゲット定義を参照して調査:

1. ターゲットがAI（ChatGPT / Claude / Perplexity）に聞きそうな質問を推定
   - ターゲットの悩み・心理状態から逆算
   - 業種×地域×目的の組み合わせで質問パターンを生成
   - 例: 「名古屋でおすすめのリフォーム業者は？」
   - 例: 「20代でベンチャーに入るメリットは？」

2. 各質問に対してAIが現在どのような情報源を参照しているか調査
   - 自サイトがAI回答に含まれる可能性の評価

3. AIに拾われるためのコンテンツ要件分析
   - Q&A形式の有効性
   - 出典付きデータの重要性
   - 独自性のあるコンテンツとは何か

出力: AI質問パターン一覧、AI回答に含まれるための戦略
```

## Phase 4: 設定ファイル更新

監査完了後、以下を `docs/seo-config.md` に追記・更新する:

1. `last_audit` の日付を更新
2. 「ターゲットのAI質問パターン（推定）」セクションに Agent 5 の結果を追記
3. 「監査履歴」テーブルに今回の結果を追加

---

## Phase 5: 次のアクション提案

レポート出力後、ユーザーに提案する:

- コンテンツ追加提案の中から次に作るコンテンツを選定
- 技術SEOの修正を実装するか確認
- コンテンツ管理パイプライン（Markdown + SSG等）の構築が必要か判断

---

## 注意事項

- **読み取り専用**: この監査はコードの変更を行わない（調査・レポートのみ）
- **設定ファイル依存**: 全ての監査判断は `docs/seo-config.md` のターゲット・方針に基づく
- **WebSearch必須**: Agent 4, 5 は `subagent_type: "general-purpose"` を使う
- **フォアグラウンド実行**: WebSearch系エージェントはバックグラウンドで実行しない（権限プロンプトが表示されないため）
- **出典重視**: キーワード調査・競合分析のデータには出典を付ける
